# คู่มือทดสอบ Demo (ครั้งเดียวจบ)

**URL:** https://vizzel-track-demo-boss.fly.dev/  
**Login:** `admin@demo.local` / `demo1234`

## 0. เตรียมตัว

- Hard refresh (Ctrl+Shift+R) หลัง deploy
- สินทรัพย์ชุด demo: **410-00-2222** มี 3 RFID ชิ้นส่วน (ใช้สแกนคืน)
- องค์กร 1 = ต้นทาง, องค์กร 2 = ปลายทาง (โอนข้ามหน่วยงาน)

---

## 1. การแจ้งเตือน (R3)

1. กระดิ่งมุมขวา → `/notifications`
2. **จัดการองค์กร → ตั้งค่าการแจ้งเตือน** → เปิด Webhook/LINE แล้วกดทดสอบ
3. ตรวจว่ามี event `withdrawal.returned` ในรายการ (ถ้ามีช่องเลือก event)

---

## 2. ผู้อนุมัติตามขั้น

1. **จัดการองค์กร → ผู้อนุมัติตามขั้น** (`/settings/approval-delegates`)
2. กำหนด user สำหรับ `section_head`, `director` (และ chief/mayor ถ้าทดสอบสาย B)
3. คิวอนุมัติจะแสดง **อนุมัติได้** เฉพาะผู้ที่ถูก delegate (หรือ role ที่เข้าเงื่อนไขถ้ายังไม่มี delegate)

---

## 3. แจ้งซ่อม (R4)

1. `/repair` → สร้างงานซ่อมจากสินทรัพย์
2. **ส่งอนุมัติ** → `/approval-queue` อนุมัติทีละขั้น (สาย A = จบที่ director)
3. ปิดงานซ่อม — ถ้าสินทรัพย์/ชิ้นส่วนยัง **ถูกยืม** ระบบต้อง **บล็อก** ไม่ให้ปิด

---

## 4. เบิก-ยืม + QR + คืนสแกน (R5)

1. สร้างคำขอเบิก → อนุมัติครบ → **ออกให้ยืม** (ได้ `issue_token` / QR)
2. เปิด `/withdrawal/verify/{token}` (หรือลิงก์จาก QR) ต้องเห็นสถานะ "อยู่ระหว่างการยืม"
3. รายการเบิก → สถานะ `borrowed` → **คืน (สแกน RFID)**
4. ชุด 410-00-2222: สแกนครบ 3 tag แล้วคืนสำเร็จ → สินทรัพย์ว่าง
5. (ถ้ามี cron) แจ้งเตือนก่อนคืน 1 วัน — event `withdrawal.due_soon`

---

## 5. โอนข้ามหน่วยงาน (R5 + R6)

1. `/transfer` → โอนจาก org 1 → org 2 → ส่งอนุมัติต้นทาง
2. อนุมัติที่ `/approval-queue` (workflow **โอนย้าย (ต้นทาง)**)
3. สลับ org ปลายทาง (หรือ user org 2) → คิว **รับโอน (ปลายทาง)** `transfer_receive`
4. อนุมัติครบ → สถานะ `pending_target` → กด **รับโอน (ยืนยัน)** ที่ `/transfer`
5. รายการขาเข้าแสดง `pending_target_approval` ระหว่างรออนุมัติปลายทาง

---

## 6. มุมมองส่วนกลาง (R6)

- `/assets/central` → เปิด "รวมหน่วยงานลูก" ดูสินทรัพย์ org ลูก

---

## 7. RFID Audit (R2)

- งานตรวจนับ → สแกน RFID ชิ้นส่วน ตรงกับ `tab_asset_component`
